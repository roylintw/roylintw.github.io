[信用卡推薦系統]使用 OpenAI 的 API

By 小書僮 13 July 2025

進入官網

OpenAI 官網

註冊帳號

新增金鑰

登入後，看到左側選單
API keys > Create new secret key

輸入資訊:

  1. Name: 隨意
  2. Project: 選擇 Default project
  3. Permissions: All 輸入完成後，點擊「Create secret key」

    金鑰就產生囉!
    這組金鑰未來用在程式呼叫 API 使用~


    這時，我習慣會在專案底下新增一個 txt 儲存金鑰，避免之後忘記，
    就把剛剛的金鑰 copy & paste

首次測試 API

打 API 後的回應訊息，429 TOO_MANY_REQUESTS

帳戶顯示預算有5美金，為什麼打 API 回覆 429 TOO_MANY_REQUESTS?


到帳單查看
Billing > Overview
Credit remaining 為 0
應該就是這個才導致無法成功打 API，
後來我參考了這篇文章別再花冤枉錢！OpenAI 免費 API 教學，額度申請全攻略（新手必看）
需要5美金才能激活帳戶，
所以我就先儲值拉~

新增付款方式

Billing > Add Payment details > Continue

Add Payment details



郵務業務- 中文地址英譯

Configure payment

  1. Initial credit purchase: 5
  2. 取消勾選 Yes, automatically recharge my card when my credit balance falls below a threshold
  3. 點擊「Continue」

Payment summary


會有額外的 5% 稅金，0.25美金，
確認正確後，點擊「Confirm payment」， 開始噴錢囉~ 付款完成後，帳戶就有5美金囉!

再次確認 Usage，就變成 $120 了


總共花了 154台幣，換 5.25美金，匯率約 29.33

啟用「資料共享（Data controls）」，領取每日免費額度

因為一開始測試，總是花費最多的時候，
只要開始「資料共享（Data Controls）」，與 OpenAI 分享你的輸入和輸出，
就可以獲得許多免費的額度!
Data controls > Sharing > Enabled for all projects > Save

勾選「By checking this box, I understand and agree to the Content Sharing Agreement above.」 > Submit

測試

我又打了一次 API，這次就有成功取得 OpenAI 解析的內容了

解析的網頁: 銀色之愛信用卡 - 台灣企銀 接著到 Usage 查看這次的使用量，
發現這次共花了 8493 個 tokens!
